    Introduction

    Welcome to the WEConnect API! You can use this API to access our endpoints, which can get information on various businesses, what users are saying about them and their average rating.

    You can connect to our API by making an HTTP request in any language of choice, and you get a JSON response.

    Requests & Responses

    Requests

    All routes in this api uses four HTTP VERBS

    • GET
    • POST
    • PUT
    • DELETE

    Your can send data with any of these to any of our routes, data sent depends on the route you are pointing to. Data sent can be in three ways

    • BODY (Form Data mostly sent with POST or PUT)
    • QUERY STRING e.g. api/v1/users?name=Ben, value is Ben, key is name
    • PARAMS e.g. route api/v1/users/:id will be accessed as api/v1/users/1 value 1 is replacing id

    Through out this doc we will always state the form of sending any data either it be as body or query string or parameter

    Responses

    We follow the Jsend Response Format for our response. Response from our API will always consist of

    • STATUS
    • BODY

    Our API uses the following status codes:

    Status Code Meaning
    200 Request OK -- Your request is valid and processes were successfull.
    201 Created -- Returned when new resource is created
    404 Not Found -- The specified resource could not be found.
    422 Unprocessable Entity -- Returned for validation errors

    Response Body

    Success response: 2XX status range

    
{
  status: "success",
  data: {
    user: {
      id: 1,
      name: "John David",
      phone: "08164488989",
      email: "daveholuborhee@gmail.com"
    }
  }
}

    Fail response: 4XX status range

    
{
  status: "fail",
  data: {
    email: "Email should be unique"
  }
}

    Error response: 5XX status range

    
{
  status: "error",
  message: "An internal server error occured"
}

    The response body is always in json format, it has three keys

    • status This should not be confused with the status code on data
    • data
    • message

    Only two of this will be returned at any point, it's either "status and data" or "status and message".

    NOTE: data will sometimes be null, even when status is "success", e.g. when deleting a business or when getting all businesses under a category which has no business under it yet.

    Authentication

    There are two routes for authentication

    • Login
    • SignUp

    Signup

    201 Success response:

    
{
  status: "success",
  data: {
    user: {
      id: 1,
      name: "John David",
      phone: "08164488989",
      email: "daveholuborhee@gmail.com"
    }
  }
}

    422 Failure Response for missing required variable example for email and phone:

    {
  status: "fail",
  data: {
    phone: "Phone is required",
    email: "Email is required",
  }
}

    422 Failure Response for password not matching:

    {
  status: "fail",
  data: {
    confirmPassword: "Password doesn't match"
  }
}

    HTTP Request

    POST https://weconnect-ng.herokuapp.com/api/v1/signup

    Request Data

    Key Type Default Required Description
    name body null true This is the name of the user to be registered
    phone body null true Contact Phone number for user.
    email body null true Contact Email Address for user.
    password body null true The Password for subsequent authentication.
    confirmPassword body null true Must match Password

    Log in

    200 Success response:

    
{
  status: "success",
  data: {
    user: {
      id: 1,
      name: "John David",
      phone: "08164488989",
      email: "daveholuborhee@gmail.com"
    }
  }
}

    422 Failure Response for missing email and phone

    {
  status: "fail",
  data: {
    email: 'There was no email in request', 
    phone: 'There was no phone in request'
  }
}

    422 Failure Response for wrong email

    {
  status: "fail",
  data: {
    email: "email@server.com doesn't match any email on server", 
  }
}

    422 Failure Response for wrong phone

    {
  status: "fail",
  data: {
    phone: "081XXXXXX doesn't match any phone on server", 
  }
}

    422 Failure Response for wrong password or missing password

    {
  status: "fail",
  data: {
    password: '12345 isn't matching for the user'
  }
}

    HTTP Request

    POST https://weconnect-ng.herokuapp.com/api/v1/signup

    Request Data

    Key Type Default Required Description
    phone body null false Contact Phone Number of user Will only be used when email is not provided.
    email body null false Contact Email Address for user Takes precedence over phone.
    password body null true The Password for subsequent authentication.

    HTTP Request

    • no filter GET https://weconnect-ng.herokuapp.com/api/v1/businesses

    • filter by location GET https://weconnect-ng.herokuapp.com/api/v1/businesses?location=owerri

    • filter by category GET https://weconnect-ng.herokuapp.com/api/v1/businesses?category=1

    • search by name GET https://weconnect-ng.herokuapp.com/api/v1/businesses?q=and

    • you can even mix the query parameters in any order GET https://weconnect-ng.herokuapp.com/api/v1/businesses?q=and&category=1&location=ketu

    Request Data

    Key Type Default Required Description
    location query null false Name of location to use for filter.
    category query null false Category Id to filter businesses.
    q query null false String value will be looked up in businesses' name.
